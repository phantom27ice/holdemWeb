# DELIVERY_ACCEPTANCE_MATRIX.md

> 目的：在交付前使用统一清单验证“可玩性 + 规则一致性 + UI 联动”。

## 使用方式

1. 先执行自动化回归：`npm test`、`npm run test:regression`、`npm run build`。
2. 启动本地预览：`npm run dev`。
3. 按下表逐项执行并记录结果（`Pass/Fail` + 备注）。

## 手工验收矩阵（12 场景）

| ID | 场景 | 操作步骤（简） | 预期结果 | 自动化覆盖 |
|---|---|---|---|---|
| M-01 | 开局与强制盲注 | 刷新进入首手，观察第一圈 | 正确发手牌；SB/BB 正确扣款；行动顺序正确 | 部分 |
| M-02 | BB option 保留 | 所有人 limp 到 BB | BB 有 `CHECK` 与 `RAISE` 选项 | 是 |
| M-03 | 最小加注边界 | 首个可行动位尝试低于最小加注 | 非法加注被拒绝；状态不推进 | 是 |
| M-04 | 弃牌直胜 | 仅剩一人未弃牌 | 直接进入派奖；赢家获主池 | 是 |
| M-05 | 四条街 check 到摊牌 | 多人持续 check 到河牌后 | 正确进入 `SHOWDOWN/PAYOUT`；筹码守恒 | 是 |
| M-06 | 单次 short all-in 不重开 | 已有下注后，短筹码全下但不足完整加注 | 前位玩家无 `RAISE` 选项 | 是 |
| M-07 | 累计 short all-in 可重开 | 多个 short all-in 累计达到完整加注 | 前位玩家重新获得 `RAISE` 选项 | 是 |
| M-08 | 边池分层与逆序结算 | 制造 3 人不同全下额度 | 先结侧池后结主池；赢家与金额正确 | 是 |
| M-09 | odd chip 分配 | 制造平分无法整除筹码场景 | 奇数筹码给按钮左侧最近赢家 | 是 |
| M-10 | 未跟注返还 | 形成未跟注金额 | 未跟注金额返还发起者 | 是 |
| M-11 | 多手自动循环 | 连续完成多手，不刷新页面 | 自动开新手；庄位轮转；筹码延续 | 是 |
| M-12 | UI 动画与反馈闭环 | 执行 fold/call/raise + 发牌/派奖 | 动作气泡、toast、pot/board/chip 动画均触发；无事件积压 | 部分 |
